## **编译过程中的"undefined reference to" 链接问题**

### 1. 链接时缺失了相关目标文件(*.o文件).
### 2. 链接时缺少相关的库文件.
### 3. 链接的库文件使用了另一个没参与编译的库文件.
### 4. 多个库文件链接顺序问题.
### 5. 定义与实现不一样.
### 6. 在C++代码中链接c语言的库，没有使用__cplusplus宏.
### 7. 目标文件中没有相应定义实现
- 验证方法：对目标文件或者库执行 nm libxxx | grep funxxx  
    1. 正常情况下一般会输出： 
    ```sh
    # U  funsss
    # T  funsss
    # 
    # U: 表示声明 
    # T：表示实现 [代码段]
    ```
