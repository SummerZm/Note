## **一句更新SQL运行时,mysql做了什么事？**

### **relog 和 binlog**
> **数据的一致性是一个大话题：事务隔离确保了业务逻辑间数据的一致性；binlog和relog的二阶段提交确保了多库间的一致性**  
1. relog: 随机IO变成顺序IO，引擎层
2. binglog: 整库备份 + binlog可实现秒级宕机恢复，应用层
3. relog 和 binlog 的二阶段提交：确保两份数据状态一致，确保主数据和多个读数据库的一致性
